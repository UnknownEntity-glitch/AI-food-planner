Система создания меню для домохозяйки

Разработать систему, которая помогает пользователям составлять планы питания на день или неделю. Система должна учитывать индивидуальные предпочтения (вкусы, аллергии, диетические цели), бюджет и, опционально, имеющиеся в наличии продукты.

Основной функционал (базовые требования):

Пользовательский интерфейс: Реализовать текстовый интерфейс (веб-страница или чат-бот), через который пользователь может вводить свои предпочтения.
База рецептов (RAG): Подготовить базу данных рецептов и реализовать RAG-систему для поиска подходящих блюд на основе запросов пользователя.
Планирование меню: Система должна уметь генерировать сбалансированное меню на заданный период (день/неделя), учитывая калорийность, БЖУ и указанные пользователем ограничения (например, «без глютена», «вегетарианское»).
Генерация списка покупок: На основе сгенерированного меню система должна формировать список продуктов, необходимых для покупки.
Пути развития и усложнения проекта (для получения максимальных баллов):

Для прикладного трека:
Агентная система: Внедрить агента, который будет обрабатывать сложные пользовательские сценарии. Например, адаптировать предложенное меню, если пользователь хочет заменить блюдо или у него внезапно появился какой-то продукт.
Мультиагентная система: Разработать систему из нескольких агентов: один агент анализирует запрос пользователя и его ограничения, второй подбирает рецепты из базы (RAG), а третий составляет из них сбалансированный план и генерирует список покупок.
Учет домашних запасов: Реализовать функцию, позволяющую пользователю ввести список имеющихся продуктов, чтобы система при планировании меню отдавала приоритет блюдам, которые можно из них приготовить. Этот функционал будет оцениваться в рамках пункта "Обработка сложных запросов с неполными данными".
Локальный инференс: Обеспечить работу системы с моделью, запущенной локально (без использования API).
Дообучение модели: Дообучить модель на данных о рецептах и принципах диетологии для генерации более качественных и разнообразных меню.
Анализ аналогов и ЦА: Провести и представить анализ существующих приложений для планирования питания, определить целевую аудиторию (например, спортсмены, семьи с детьми, люди с аллергиями) и обосновать полезность своего решения.
